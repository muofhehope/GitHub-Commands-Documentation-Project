A Git Commit:

Saves your changes to the local Git repository as a new version in the project's history.

How Do I Commit?

After you've entered a message, save the file and close the editor to create the actual commit.

  1. git commit -a.
  2. git commit -m "commit message"
  3. git commit -am "commit message"
  4. git commit --amend.

Here are some examples of how to use the git commit command: Basic commit with a message.

git commit -m "Fix: Resolve issue with user authentication"
git commit -a -m "initial commit"

What Git commit Really Does:

Git commit is a command in Git that saves a snapshot of your changes to the local repository. It marks a specific point in your project's history that you can go back to later. Think of it like saving a version of your project with a message describing what changed and why.

Git Fork:

A fork is a personal copy of someone else's repository that lives on your GitHub account. It lets you freely experiment with changes without affecting the original project (called the "upstream" repo).

Typical Fork Workflow (Syntax Style)

Step 1: Fork on GitHub

Do this through the GitHub interface:

  • Go to the repo you want to fork (e.g., https://github.com/original/repo)
  • Click the "Fork" button in the top-right corner

This Create:

https://github.com/yourusername/repo

Step 2: Clone Your Fork

git clone https://github.com/yourusername/repo.gitcd repo

Step 3: Add the Original Repo as a Remote

git remote add upstream https://github.com/original/repo.git
  • origin = your fork (on your GitHub)
  • upstream = the original repo (where you forked from)

Step 4: Sync with the Original Repo

To get new changes from the original project:

  1. git fetch upstream
  2. git checkout main
  3. git merge upstream/main

Or if the original repo uses master:

  1. git fetch upstream
  2. git checkout master
  3. git merge upstream/main

Step 5: Push Changes to Your Fork

git push origin main

Git Merge:

When you‚Äôre working on a project with multiple branches (for features, bug fixes, experiments, etc.), you‚Äôll eventually want to bring those changes back together. git merge lets you do that safely, preserving the history of both branches.

Steps to Use git merge:

2. Make sure main is up to date

What Is A Git merge?

Git merge is a Git command that combines changes from one branch into another. It integrates the histories of two branches and produces a new merge commit if necessary.

Git Branch:

The git branch command in Git is used to create, list, rename, and delete branches in your repository.

Typical Workflow:

1. Create a new branch:

Git Branch New-feature

2. Switch to it:

git checkout new-feature

3. Make changes, commit, and merge back when done:

  1. git checkout main
  2. git merge new-feature

What is Git branch?:

The git branch command is used to work with branches in Git.

A branch is like a separate workspace where you can make changes without affecting the main project. This is useful when you're working on:

  • A new feature
  • A bug fix
  • An experiment

Git Push:

The git push command is used to upload your local Git commits to a remote repository (like GitHub, GitLab, or Bitbucket).

What Does Git push Do?:

git push origin main

It tells Git:

"Take my commits from the local main branch and send them to the main branch on the remote called origin."

In Short:

git push = Send your changes to the cloud (remote repo).

Why Use Git push?:

  • Share your work with teammates
  • Back up your code to a remote server
  • Publish changes for deployment or collaboration
  • Submit contributions (e.g., after forking a repo)

Git Branch Plus:

A Git branch is used to list, create, rename, and delete branches in Git. A branch in Git represents an independent line of development, allowing you to work on features, bug fixes, or experiments without affecting the main codebase.

Why Use git push?:

  • Share your work with others
  • Publish updates to a shared project
  • Deploy code (in some setups)
  • Back up your changes remotely

Basic Syntax:

git push

Example:

git push origin main

This pushes your local main branch to the remote named origin.

What is a Pull Request?:

A pull request is:

  • A proposal to merge your changes (commits) into a project
  • Usually created after you push a feature branch to a remote repository
  • Often used in team collaboration and open-source contributions
  • A proposal to merge your changes (commits) into a project

Pull Request Workflow:

1. Fork or clone the repository:

git clone https://github.com/example/project.git

2. Create a new branch:

git checkout -b feature-login

3. Make changes and commit:

  1. git add .
  2. git commit -m "Add login feature"

4. Push your branch to remote:

git push origin feature-login

5. Open a pull request:

  1. Go to the repository on GitHub/GitLab
  2. Click "Compare & pull request" (GitHub)
  3. Add a title and description
  4. Submit for review
